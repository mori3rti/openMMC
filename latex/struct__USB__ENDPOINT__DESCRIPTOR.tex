Interval for polling endpoint for data transfers. Expressed in frames or microframes depending on the device operating speed (i.\-e., either 1 millisecond or 125 µs units). \begin{DoxyItemize}
\item For full-\//high-\/speed isochronous endpoints, this value must be in the range from 1 to 16. The b\-Interval value is used as the exponent for a $ 2^(bInterval-1) $ value; e.\-g., a b\-Interval of 4 means a period of 8 ( $ 2^(4-1) $). \item For full-\//low-\/speed interrupt endpoints, the value of this field may be from 1 to 255. \item For high-\/speed interrupt endpoints, the b\-Interval value is used as the exponent for a $ 2^(bInterval-1) $ value; e.\-g., a b\-Interval of 4 means a period of 8 ( $ 2^(4-1) $) . This value must be from 1 to 16. \item For high-\/speed bulk/control O\-U\-T endpoints, the b\-Interval must specify the maximum N\-A\-K rate of the endpoint. A value of 0 indicates the endpoint never N\-A\-Ks. Other values indicate at most 1 N\-A\-K each b\-Interval number of microframes. This value must be in the range from 0 to 255. \par
Refer to Chapter 5 of U\-S\-B 2.\-0 specification for more information. \end{DoxyItemize}

This field describes the endpoint’s attributes when it is configured using the b\-Configuration\-Value. \par
 Bits 1..0\-: Transfer Type \begin{DoxyItemize}
\item 00 = Control \item 01 = Isochronous \item 10 = Bulk \item 11 = Interrupt \par
 If not an isochronous endpoint, bits 5..2 are reserved and must be set to zero. If isochronous, they are defined as follows\-: \par
 Bits 3..2\-: Synchronization Type \item 00 = No Synchronization \item 01 = Asynchronous \item 10 = Adaptive \item 11 = Synchronous \par
 Bits 5..4\-: Usage Type \item 00 = Data endpoint \item 01 = Feedback endpoint \item 10 = Implicit feedback Data endpoint \item 11 = Reserved \par
 Refer to Chapter 5 of U\-S\-B 2.\-0 specification for more information. \par
 All other bits are reserved and must be reset to zero. Reserved bits must be ignored by the host. \par

Maximum packet size this endpoint is capable of sending or receiving when this configuration is selected. \par
For isochronous endpoints, this value is used to reserve the bus time in the schedule, required for the per-\/(micro)frame data payloads. The pipe may, on an ongoing basis, actually use less bandwidth than that reserved. The device reports, if necessary, the actual bandwidth used via its normal, non-\/\-U\-S\-B defined mechanisms. \par
For all endpoints, bits 10..0 specify the maximum packet size (in bytes). \par
For high-\/speed isochronous and interrupt endpoints\-: \par
Bits 12..11 specify the number of additional transaction opportunities per microframe\-: \par
\begin{DoxyItemize}
\item 00 = None (1 transaction per microframe) \item 01 = 1 additional (2 per microframe) \item 10 = 2 additional (3 per microframe) \item 11 = Reserved \par
Bits 15..13 are reserved and must be set to zero. \end{DoxyItemize}
