00101 /******************************************************************************
00102  * TRACE_RECORDER_STORE_MODE
00103  *
00104  * Macro which should be defined as one of:
00105  * - TRACE_STORE_MODE_RING_BUFFER
00106  * - TRACE_STORE_MODE_STOP_WHEN_FULL
00107  * Default is TRACE_STORE_MODE_RING_BUFFER.
00108  *
00109  * With TRACE_RECORDER_STORE_MODE set to TRACE_STORE_MODE_RING_BUFFER, the 
00110  * events are stored in a ring buffer, i.e., where the oldest events are 
00111  * overwritten when the buffer becomes full. This allows you to get the last
00112  * events leading up to an interesting state, e.g., an error, without having 
00113  * to store the whole run since startup.
00114  *
00115  * When TRACE_RECORDER_STORE_MODE is TRACE_STORE_MODE_STOP_WHEN_FULL, the 
00116  * recording is stopped when the buffer becomes full. This is useful for
00117  * recording events following a specific state, e.g., the startup sequence.
00118  *****************************************************************************/
00119 #define TRACE_RECORDER_STORE_MODE TRACE_STORE_MODE_STOP_WHEN_FULL

00377 /******************************************************************************
00378  * USE_IMPLICIT_IFE_RULES
00379  *
00380  * Macro which should be defined as either zero (0) or one (1). 
00381  * Default is 1.
00382  *
00383  * Tracealyzer groups the events into actor instances, based on context-switches
00384  * and a definition of "Instance Finish Events", or IFEs. These are kernel calls 
00385  * considered to be the last event in a task instance. Some kernel calls are 
00386  * considered IFEs by default (e.g., delay functions), but it is also possible
00387  * to specify this individually for each task (see vTraceTaskInstanceFinish).
00388  *
00389  * If USE_IMPLICIT_IFE_RULES is one (1), the default IFEs will be enabled, which
00390  * gives a "typical" grouping of events into instances. You can combine this 
00391  * with calls to vTraceTaskInstanceFinish for specific tasks.
00392  *
00393  * If USE_IMPLICIT_IFE_RULES is zero (0), the implicit IFEs are disabled and all
00394  * events withing each task is then shown as a single instance, unless  you call 
00395  * vTraceTaskInstanceFinish() at suitable locations to mark the IFEs.
00396  *****************************************************************************/
00397 #define USE_IMPLICIT_IFE_RULES 1

00399 /******************************************************************************
00400  * USE_16BIT_OBJECT_HANDLES
00401  *
00402  * Macro which should be defined as either zero (0) or one (1).
00403  * 
00404  * If set to 0 (zero), the recorder uses 8-bit handles to identify kernel 
00405  * objects such as tasks and queues. This limits the supported number of
00406  * concurrently active objects to 255 of each type (object class).
00407  *
00408  * If set to 1 (one), the recorder uses 16-bit handles to identify kernel 
00409  * objects such as tasks and queues. This limits the supported number of
00410  * concurrent objects to 65535 of each type (object class). However, since the
00411  * object property table is limited to 64 KB, the practical limit is about
00412  * 3000 objects in total. 
00413  * 
00414  * Default is 0.
00415  *
00416  * NOTE: An object with handle above 255 will use an extra 4-byte record in 
00417  * the event buffer whenever referenced. Moreover, some internal tables in the 
00418  * recorder gets larger when using 16-bit handles. The additional RAM usage is 
00419  * 5-10 byte plus 1 byte per kernel object i.e., task, queue, mutex, etc.
00420  *****************************************************************************/
